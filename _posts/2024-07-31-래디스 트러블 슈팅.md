# 원인 파악
* 레디스 에러 메시지에 주목했다.
```sh
io.lettuce.core.redisCommandExecutionException: MISCONF redis is configured to save RDB snapshots, but it's currently unable to persist to disk. Commands that may modify the data set are disabled, because this instance is configured to report errors during writes if RDB snapshotting fails (stop-writes-on-bgsave-error option). Please check the redis logs for details about the RDB error.
```
* 해석 : RDB 스냅샷을 저장하게 설정되어있어. 그런데 디스크에 영구저장이 지금 불가능 하네? ㅠ 저 설정을 확인해.
* 구글링도 해보니, 이런 글을 찾았다.
* https://velog.io/@guswns3371/redis-MISCONF-redis-is-configured-to-save-RDB-snapshots-but-it-is-currently-not-able-to-persist-on-disk
* 일단 디스크 저장기능을 안쓰고 우리는 캐싱 용도로만 쓰고 있다.
* 디스크 저장 기능을 끄자!


# 해결
* 일단 디스크 기능을 안쓰고 우리는 캐싱 용도로만 쓰고 있다.
* 따라서 stop-writes-on-bgsave-error option 을 아래와 같이 설정했다.
```sh
onfig set stop-writes-on-bgsave-error no
```

# 예방
* 사이드 프로젝트는 docker 를 이용하여 배포하고 있다.
* 따라서 레디스 이미지를 이용하는 코드 부분에 아래 코드를 추가하여 다음과 같이 완성하였다.
* 앞으로 재배포를 했을때 동일 현상이 발생하지 않게 되었다.
```sh
version: '3.9'
services:
  redis:
    container_name: "redis"
    image: redis:7.2.4
    user: "1001:1001"
    logging:
      driver: "json-file"
      options:
        max-size: "10m"
        max-file: "2"
    expose:
      - "${redis_PORT}"
    ports:
      - "${redis_PORT}:6379"
    restart: always
    volumes:
      - ~/data/redis:/data
    network_mode: bridge
    deploy:
      mode: replicated
      replicas: 1
      resources:
        limits:
          cpus: '0.2'
        reservations:
          memory: 512M
    environment:
      TZ: Asia/Seoul
      redis_PASSWORD: ${redis_PASSWORD}
    command: >
      sh -c "redis-server --notify-keyspace-events gxE --requirepass $redis_PASSWORD &
             sleep 5 &&
             redis-cli -a '$redis_PASSWORD' CONFIG SET stop-writes-on-bgsave-error no"
    extra_hosts:
      - "host.docker.internal:host-gateway"
```
